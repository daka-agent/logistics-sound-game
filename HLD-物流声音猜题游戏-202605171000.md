## 2. 背景与目标

### 2.1 设计背景

本系统是一个基于Web的物流声音猜题游戏,核心玩法为"听声音猜物流场景",包含积分系统和排行榜功能。系统需支持跨平台访问(PC/移动端),并保证良好的用户体验和扩展性。

### 2.2 设计目标

**技术目标:**
- 实现轻量级、高性能的单页应用(SPA)
- 保证声音加载流畅,游戏体验无卡顿
- 支持快速迭代和功能扩展
- 降低部署和运维成本

**设计原则:**
- 前后端分离,接口清晰
- 配置与代码解耦,便于维护
- 移动优先,响应式设计
- 渐进增强,优雅降级

## 5. 核心技术选型与理由

### 5.1 前端技术选型

**Vue 3 + Composition API**

选型理由:
- 学习曲线平缓,开发效率高
- Composition API提供更好的逻辑复用和代码组织
- 响应式系统性能优异,适合游戏交互场景
- 生态成熟,社区活跃,问题易解决
- 与Vite深度集成,开发体验极佳

**Element Plus**

选型理由:
- 组件丰富,覆盖常用UI需求
- 样式统一,设计规范完善
- 支持按需引入,打包体积可控
- 中文化完善,无需额外配置

**Pinia**

选型理由:
- Vue3官方推荐,TypeScript支持完善
- 模块化设计,状态管理清晰
- 开发工具集成,调试方便
- 比Vuex更简洁,学习成本低

**HTML5 Audio API**

选型理由:
- 浏览器原生支持,无需额外依赖
- 控制灵活,可精确控制播放
- 兼容性好,移动端支持完善
- 性能优秀,无额外开销

### 5.2 后端技术选型

**Node.js + Express**

选型理由:
- JavaScript全栈,技术栈统一
- 异步I/O性能优秀,适合并发场景
- 中间件生态丰富,快速搭建
- 部署简单,运维成本低
- 与前端开发技能复用

**SQLite**

选型理由:
- 轻量级,无需独立数据库服务
- 单文件存储,备份迁移简单
- 读写性能足够(日访问量<10万)
- 零配置,降低运维成本
- 支持SQL,数据操作灵活

**替代方案对比:**

| 方案 | 优势 | 劣势 | 结论 |
|------|------|------|------|
| MongoDB | 文档灵活,适合JSON数据 | 需独立服务,运维成本高 | 不适合小规模场景 |
| MySQL | 性能强大,生态成熟 | 需独立服务,配置复杂 | 过度设计 |
| SQLite | 轻量零配置 | 并发写入有限 | **推荐** |

### 5.3 声音素材方案

**方案对比:**

| 方案 | 优势 | 劣势 | 选择 |
|------|------|------|------|
| 本地存储 | 加载快,无依赖 | 打包体积大,难以更新 | 不采用 |
| CDN托管 | 全球加速,易更新 | 需付费CDN服务 | 备选 |
| 静态资源 | 简单直接,无依赖 | 首次加载较慢 | **推荐** |

**最终方案:**
- 开发阶段:本地静态资源
- 生产阶段:打包为静态文件,Nginx托管
- 未来扩展:可切换至CDN(配置化)

## 6. 数据流/调用链说明

### 6.1 核心业务流程

**流程1:游戏开始**

游戏开始流程:

用户 → 点击开始
  ↓
GameStore初始化 → 重置积分/进度
  ↓
调用questionService.getQuestions()
  ↓
后端查询questions.json
  ↓
随机选取10题 → 返回题目ID列表
  ↓
前端预加载第一题音频
  ↓
跳转至游戏页面 → 开始答题

**流程2:答题过程**

答题流程:

用户选择选项
  ↓
GameStore.judgeAnswer(selectedOption)
  ↓
判断正误 → 计算积分
  ↓
显示AnswerFeedback组件
  ↓
预加载下一题音频
  ↓
1秒后 → 进入下一题
  ↓
重复流程(共10题)
  ↓
游戏结束 → 跳转结算页

**流程3:提交成绩**

成绩提交流程:

游戏结束
  ↓
收集数据(积分、用时、正确率)
  ↓
调用leaderboardService.submitScore()
  ↓
后端验证 → 防刷分检查
  ↓
写入SQLite数据库
  ↓
返回排名信息
  ↓
前端显示成绩总结

### 6.3 音频加载策略

**预加载策略:**

音频加载时序:

游戏初始化:
  - 预加载第1题音频(阻塞,保证立即可播)
  
每题答题时:
  - 播放当前题音频
  - 异步预加载下一题音频(非阻塞)
  
游戏结束时:
  - 清除未使用的音频缓存

**降级方案:**
- 音频加载超时(>5秒):显示"加载中"动画
- 音频加载失败:显示错误提示,提供跳过选项
- 弱网环境:降低音质,加载低码率版本

## 7. 关键接口定义

### 7.1 题目接口

**获取题目列表**

请求:
- 方法: GET
- 路径: /api/questions
- 参数: 
  - count: 题目数量(默认10)
  - difficulty: 难度等级(可选,1-3)

响应:
- 状态码: 200
- 数据格式:
  - success: true
  - data:
    - questions: 题目数组
      - id: 题目ID
      - audioUrl: 音频URL
      - options: 选项数组
      - difficulty: 难度

**获取题目详情**

请求:
- 方法: GET
- 路径: /api/questions/:id
- 参数: 无

响应:
- 状态码: 200
- 数据格式:
  - success: true
  - data:
    - id: 题目ID
    - audioUrl: 音频URL
    - correctAnswer: 正确答案
    - options: 选项数组
    - description: 声音描述
    - scene: 场景说明
    - difficulty: 难度

### 7.2 排行榜接口

**提交成绩**

请求:
- 方法: POST
- 路径: /api/leaderboard
- 参数:
  - nickname: 昵称(1-20字符)
  - score: 积分
  - correctRate: 正确率
  - timeUsed: 用时(秒)

响应:
- 状态码: 200
- 数据格式:
  - success: true
  - data:
    - rank: 当前排名
    - isRecord: 是否破纪录

**获取排行榜**

请求:
- 方法: GET
- 路径: /api/leaderboard
- 参数:
  - type: 排行类型(weekly/monthly)
  - limit: 返回数量(默认10)

响应:
- 状态码: 200
- 数据格式:
  - success: true
  - data:
    - leaderboard: 排行数组
      - rank: 排名
      - nickname: 昵称
      - score: 积分
      - correctRate: 正确率
      - createdAt: 时间
    - userRank: 当前用户排名(如登录)

### 7.3 统计接口

**提交游戏统计**

请求:
- 方法: POST
- 路径: /api/stats
- 参数:
  - score: 积分
  - correctRate: 正确率
  - timeUsed: 用时
  - completed: 是否完成(中途退出标记)

响应:
- 状态码: 200
- 数据格式:
  - success: true

**获取统计概览**

请求:
- 方法: GET
- 路径: /api/stats/overview
- 参数: 无

响应:
- 状态码: 200
- 数据格式:
  - success: true
  - data:
    - totalGames: 总游戏次数
    - avgScore: 平均积分
    - avgCorrectRate: 平均正确率
    - activeUsers: 今日活跃用户

### 7.4 统一响应格式

成功响应:
- success: true
- data: 响应数据
- message: 提示信息(可选)

失败响应:
- success: false
- error:
  - code: 错误码
  - message: 错误信息
- data: null

## 8. 非功能设计

### 8.1 性能设计

**前端性能优化:**

| 优化项 | 实现方式 | 预期效果 |
|--------|---------|---------|
| 代码分割 | Vue Router懒加载 | 首屏加载减少50% |
| 资源压缩 | Vite生产构建压缩 | 体积减少60% |
| 音频预加载 | 异步预加载下一题 | 切换无延迟 |
| 图片懒加载 | Intersection Observer | 首屏加载加速 |
| 缓存策略 | Service Worker | 二次访问秒开 |

**后端性能优化:**

| 优化项 | 实现方式 | 预期效果 |
|--------|---------|---------|
| 数据库索引 | 排名字段建立索引 | 查询加速10倍 |
| 连接池 | SQLite连接复用 | 减少连接开销 |
| 响应压缩 | Gzip中间件 | 传输减少70% |
| 缓存层 | 内存缓存排行榜 | 减少数据库查询 |

**性能指标:**

| 指标 | 目标值 | 监控方式 |
|------|--------|---------|
| 首屏加载 | ≤ 3秒 | Lighthouse |
| 音频加载 | ≤ 2秒 | Performance API |
| 接口响应 | ≤ 500ms | 后端日志 |
| 内存占用 | ≤ 100MB | 浏览器工具 |

### 8.2 可用性设计

**容错机制:**

- 前端:
  - 音频加载失败重试(最多3次)
  - API请求失败降级(本地数据)
  - 路由错误跳转404页面
  - 组件错误边界捕获

- 后端:
  - 数据库连接失败重连
  - 异常请求返回友好错误
  - 进程守护(PM2自动重启)

**监控告警:**

- 错误日志记录(Winston)
- 接口响应时间监控
- 异常流量告警(请求限流触发)

### 8.3 可扩展性设计

**水平扩展:**

- 前端:静态资源可部署至CDN,无状态易扩展
- 后端:无状态服务,可部署多实例(需切换数据库)

**垂直扩展:**

- 数据库:SQLite → PostgreSQL/MySQL(无需改代码)
- 缓存:可引入Redis缓存排行榜

**功能扩展:**

| 功能 | 扩展点 | 实现方式 |
|------|--------|---------|
| 难度模式 | QuestionService | 增加难度筛选逻辑 |
| 好友对战 | WebSocket | 引入Socket.io |
| 每日挑战 | TaskService | 定时任务生成题库 |
| 多语言 | i18n | vue-i18n插件 |

### 8.4 安全设计

**前端安全:**

- XSS防护:Vue自动转义,v-html慎用
- CSRF防护:请求携带Token
- 敏感数据:不存储用户隐私,仅存匿名ID

**后端安全:**

- 输入验证:所有请求参数校验
- SQL注入:使用参数化查询
- 请求限流:同一IP限频(100次/分钟)
- 敏感词过滤:昵称过滤敏感词
- 防刷分:单用户每日提交限制(10次/天)

**安全清单:**

- [ ] 所有API使用HTTPS
- [ ] 生产环境关闭调试接口
- [ ] 定期更新依赖修复漏洞
- [ ] 数据库文件权限控制
- [ ] 日志不记录敏感信息
