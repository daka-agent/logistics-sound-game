## 2. 背景与目标

### 2.1 项目背景

物流行业涉及大量机械设备和作业流程,每种操作都会产生独特的声音特征。例如:
- 运输车辆:卡车引擎声、倒车提示音、气刹声
- 装卸设备:叉车引擎声、液压升降声、托盘碰撞声
- 分拣系统:传送带运转声、扫码枪滴声、包裹滑落声
- 仓储设施:集装箱门开合声、货架移动声、堆高机作业声

这些声音具有鲜明的职业特征,但普通人往往难以准确识别。本项目旨在通过游戏化方式,让用户在娱乐中了解物流行业的声音特征。

### 2.2 项目目标

**核心目标:**
- 创建一个有趣的"听声音猜题"Web游戏
- 通过游戏传播物流行业知识
- 提供简单的积分系统和排行榜增强粘性

**成功指标:**
- 用户平均游戏时长 ≥ 5分钟
- 用户完成游戏局数 ≥ 3局/访问
- 7日留存率 ≥ 30%

## 3. 用户角色与场景

### 3.1 用户角色

| 角色类型 | 描述 | 典型特征 |
|---------|------|---------|
| 普通玩家 | 对物流好奇的普通用户 | 无专业知识,希望娱乐消遣 |
| 物流从业者 | 物流行业工作人员 | 有一定声音识别经验,寻求职业认同感 |
| 挑战者 | 喜欢排行榜竞争的用户 | 追求高分,多次尝试 |

### 3.2 核心场景

**场景1:初次体验**
- 用户打开游戏网页
- 查看简单的新手引导
- 开始第一局游戏,听声音猜测
- 查看积分和正确答案

**场景2:持续游玩**
- 用户再次访问,看到历史积分
- 开始新一局游戏
- 查看排行榜位置
- 分享成绩到社交媒体

**场景3:好友互动**
- 用户分享游戏链接给好友
- 好友打开并游玩
- 两人在排行榜上比较成绩

### 4.2 用户故事与验收标准

**F001 - 声音播放**

用户故事:作为玩家,我希望能够清晰听到声音并重复播放,以便准确判断声音来源。

验收标准:
- [ ] 每题自动播放一次声音片段(3-8秒)
- [ ] 提供"重播"按钮,用户可点击重复播放
- [ ] 声音播放时显示播放进度条
- [ ] 音量可调节(默认80%音量)
- [ ] 声音加载失败时显示友好提示

**F002 - 选项展示**

用户故事:作为玩家,我希望看到清晰的选项列表,以便快速做出选择。

验收标准:
- [ ] 每题显示4个选项(1个正确+3个干扰项)
- [ ] 选项使用文字描述,简洁明了
- [ ] 选项随机排列顺序
- [ ] 用户选择后选项高亮显示
- [ ] 支持键盘快捷键(1/2/3/4)选择

**F003 - 答题判断**

用户故事:作为玩家,我希望立即知道答案是否正确,获得即时反馈。

验收标准:
- [ ] 选择正确时显示绿色✓,播放成功音效
- [ ] 选择错误时显示红色✗,播放失败音效
- [ ] 显示正确答案(如果选错)
- [ ] 答题后自动进入下一题(间隔1秒)

**F004 - 积分系统**

用户故事:作为玩家,我希望看到我的得分累积,获得成就感。

验收标准:
- [ ] 正确答题+10分
- [ ] 首次答对额外+5分(连续答对奖励)
- [ ] 实时显示当前积分和题号(如:第3题/共10题)
- [ ] 显示本次游戏正确率
- [ ] 游戏结束时显示总积分和历史最高分

**F005 - 答案解析**

用户故事:作为玩家,我希望了解声音的详细来源,增长物流知识。

验收标准:
- [ ] 答题后显示声音描述(如:"叉车倒车时的液压系统运转声")
- [ ] 显示声音来源场景(如:"常见于仓库装卸作业区")
- [ ] 可选展开详细说明(物流知识小卡片)

**F006 - 排行榜**

用户故事:作为挑战者,我希望看到自己的排名,与朋友比较成绩。

验收标准:
- [ ] 显示本周TOP10玩家(昵称+积分)
- [ ] 显示本月TOP10玩家
- [ ] 当前用户排名高亮显示(如在榜外显示"您排名:第XX名")
- [ ] 排行榜每局游戏结束后刷新

**F007 - 游戏流程**

用户故事:作为玩家,我希望游戏流程清晰可控,知道自己的进度。

验收标准:
- [ ] 每局游戏包含10道题
- [ ] 显示进度条(如:●●●○○○○○○○)
- [ ] 游戏中途可退出,保存当前积分
- [ ] 游戏结束显示成绩总结(总积分、正确率、用时)
- [ ] 提供"再玩一局"和"查看排行榜"按钮

**F008 - 新手引导**

用户故事:作为新玩家,我希望快速了解游戏规则,立即开始游戏。

验收标准:
- [ ] 首次访问显示3屏引导(游戏规则→操作说明→开始游戏)
- [ ] 支持跳过引导直接开始
- [ ] 引导内容简洁(每屏不超过3行文字)
- [ ] 后续访问不再显示引导(使用Cookie记录)

**F009 - 分享功能**

用户故事:作为玩家,我希望分享成绩给朋友,邀请他们一起玩。

验收标准:
- [ ] 游戏结束提供"分享成绩"按钮
- [ ] 生成分享文案(如:"我在物流声音猜题获得120分,正确率80%!快来挑战吧!")
- [ ] 支持复制链接或直接分享到微信/QQ
- [ ] 分享链接带参数(可选:发起挑战模式)

## 5. 非功能需求

### 5.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 页面加载时间 | ≤ 3秒 | 首屏加载完成 |
| 声音加载时间 | ≤ 2秒/题 | 单个声音文件加载 |
| 响应时间 | ≤ 500ms | 用户操作到界面反馈 |
| 并发支持 | ≥ 1000人 | 同时在线游玩 |

### 5.2 兼容性要求

| 类型 | 支持范围 |
|------|---------|
| 浏览器 | Chrome 90+, Safari 14+, Firefox 88+, Edge 90+ |
| 设备 | PC、平板、手机 |
| 分辨率 | 320px - 1920px 自适应 |
| 网络 | 4G/5G/WiFi,支持弱网降级 |

### 5.3 安全要求

- 不收集用户敏感信息(手机号、身份证等)
- 排行榜昵称过滤敏感词
- 防刷分机制(同一IP每日游戏次数限制)

### 5.4 可维护性要求

- 声音素材易于替换和扩展
- 题目配置与代码分离(JSON配置)
- 提供管理后台查看游戏数据统计

## 6. 交互/原型规范

### 6.1 页面结构

游戏包含4个核心页面:

**页面1:欢迎页**
- 游戏Logo和标题
- "开始游戏"按钮
- "游戏规则"链接
- 简单的背景动画(声音波形效果)

**页面2:游戏页**
- 顶部:当前积分 + 进度条
- 中间:声音播放区域(波形可视化)
- 下部:4个选项按钮(2x2网格布局)
- 底部:重播按钮 + 音量调节

**页面3:答题反馈**
- 覆盖在游戏页上方的弹窗
- 显示正确/错误状态
- 显示声音解析说明
- 1秒后自动消失进入下一题

**页面4:结算页**
- 本局成绩(积分、正确率、用时)
- 历史最高分
- 排行榜展示
- "再玩一局" + "分享成绩"按钮

### 6.2 视觉风格

**整体风格:**简洁现代,科技感

**色彩方案:**
- 主色:#0066CC(物流蓝)
- 辅色:#00A878(成功绿)、#FF4757(错误红)
- 背景:#F5F7FA(浅灰)
- 文字:#2C3E50(深灰)

**字体:**
- 标题:思源黑体 Bold 24px
- 正文:思源黑体 Regular 16px
- 选项:思源黑体 Medium 18px

**音效反馈:**
- 正确:清脆的"叮"声
- 错误:低沉的"嘟"声
- 按钮:轻微的"哒"声

## 7. 数据需求

### 7.1 题目数据结构

题目配置存储为JSON格式,每道题包含:

字段说明:
- id:题目唯一标识
- audio_url:声音文件URL
- correct_answer:正确答案
- options:4个选项数组(含正确答案)
- description:声音描述
- scene:声音场景说明
- difficulty:难度等级(1-简单,2-中等,3-困难)

示例题目:

题目示例:
- id: "Q001"
- audio_url: "/sounds/forklift-reverse.mp3"
- correct_answer: "叉车倒车提示音"
- options: ["叉车倒车提示音", "卡车倒车提示音", "传送带启动声", "扫码枪提示音"]
- description: "叉车倒车时发出的蜂鸣警示声"
- scene: "常见于仓库、配送中心装卸区域"
- difficulty: 1

### 7.2 用户数据

**本地存储(Cookie/LocalStorage):**
- 用户ID(匿名生成)

**服务端存储:**
- 排行榜数据(昵称、积分、时间)
- 游戏统计(每日游玩人次、平均得分)

## 8. 依赖与约束

### 8.1 技术依赖

| 依赖项 | 用途 | 说明 |
|--------|------|------|
| HTML5 Audio API | 声音播放 | 浏览器原生支持 |
| Vue.js 3 / React | 前端框架 | 快速开发交互界面 |
| Node.js + Express | 后端服务 | 排行榜API、统计接口 |
| MongoDB / SQLite | 数据存储 | 排行榜数据、统计数据 |
| 免费音效网站 | 声音素材 | Freesound.org、Pixabay等 |

### 8.2 约束条件

**时间约束:**
- MVP版本目标:2周内完成核心功能
- 正式版本:4周内完成全部功能

**资源约束:**
- 前端开发:1人
- 后端开发:1人(可复用前端人员)
- 设计资源:使用现成UI组件库(Ant Design / Element Plus)
- 声音素材:免费音效库(质量可能不如真实录音)

**技术约束:**
- 声音文件格式:MP3(兼容性最好)
- 单个声音文件大小:≤ 500KB(保证加载速度)
- 不支持实时多人对战(简化开发)

### 8.3 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 免费音效不够真实 | 用户体验下降 | 尝试多个音效网站,筛选最接近真实的;考虑后期补充真实录音 |
| 用户刷分作弊 | 排行榜公平性受损 | 限制单IP每日次数,记录异常行为,必要时实名验证 |
| 声音加载慢 | 用户流失 | 预加载下一题声音,显示加载动画,提供弱网降级方案 |
| 浏览器兼容问题 | 部分用户无法游玩 | 提示用户升级浏览器,提供兼容性检测页面 |
